# ⚠️ UX 엣지 케이스 및 대응 전략

AI 연동 과정에서 발생할 수 있는 예외 상황과 UI 피드백 방안을 정의합니다.

## 1. 데이터 부족 케이스
- **상황**: 선택된 커밋의 메시지가 "fix", "update" 등 매우 부실한 경우.
- **대응**: 프론트엔드에서 최소 글자 수 검증 후 경고 메시지 표시 ("선택한 커밋의 내용이 부족하여 AI가 상세한 글을 쓰기 어려울 수 있습니다. 계속하시겠습니까?").

## 2. AI 응답 지연 (Timeout)
- **상황**: Gemini API 응답이 30초 이상 지연되는 경우.
- **대응**: 
  - 10초 경과 시: "거의 다 되었습니다. AI가 내용을 다듬고 있어요..." 문구 노출.
  - 60초 초과 시: 타임아웃 에러 처리 및 "다시 시도" 버튼 제공.

## 3. 백엔드/API 에러
- **상황**: API Key 만료, Rate Limit 도달, 서버 500 에러.
- **대응**: `design.md`의 `--color-accent-blue`를 활용한 에러 배너 노출. 구체적인 에러 원인보다는 "서버와 통신이 원활하지 않습니다"라는 친절한 메시지 제공.

## 4. 로딩 UI
- **상황**: LLM 생성 중.
- **대응**: 스켈레톤 UI가 아닌, 블로그 초안이 작성되고 있음을 암시하는 타이핑 애니메이션 또는 진행 상태 바(Progress bar) 적용.
